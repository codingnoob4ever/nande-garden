# 碎片存档系统说明

## 📝 什么是碎片存档

碎片存档是楠得心灵花园的**原文记录系统**，用于保存每天所有输入的原始内容。

## 🎯 为什么需要碎片存档

1. **完整性**: 保留所有原始想法，不遗漏任何细节
2. **可追溯**: 回顾时可以看到最初的表达方式
3. **再分析**: 后续可以基于原文进行二次分析
4. **对比**: 对比原文和复盘，看到小北的加工过程

## 📂 存档结构

```
fragments/
├── 2026/
│   ├── 01-January/
│   │   ├── 2026-01-21.md
│   │   ├── 2026-01-22.md
│   │   └── ...
│   ├── 02-February/
│   └── ...
└── 2025/
```

## 📋 存档格式

每个碎片文件包含：

### 文件头

```markdown
# 碎片记录 - YYYY-MM-DD

> 本文件记录YYYY年M月D日的所有原始输入碎片

---
```

### 碎片流水

```markdown
## 📝 碎片流水

[HH:mm] 第一条碎片内容

[HH:mm] 第二条碎片内容

[HH:mm] 第三条碎片内容
```

### 统计信息

```markdown
## 📊 统计信息

- **碎片数量**: X条
- **记录时段**: HH:mm - HH:mm
- **时间跨度**: X小时
- **生成日报**: HH:mm (如已生成)
```

## ⚙️ 自动化机制

### 实时追加

- 每次收录碎片时立即写入
- 格式: `[HH:mm] 原文内容`
- 自动添加时间戳

### 文件管理

- 一天一个文件
- 自动创建日期目录
- 文件名: `YYYY-MM-DD.md`

### 存档路径

```
fragments/YYYY/MM-MonthName/YYYY-MM-DD.md
```

示例: `fragments/2026/01-January/2026-01-21.md`

## 🔍 使用场景

### 场景1: 日常记录

```
楠哥: 今天感觉状态不错
小北: 🤐 已收录
系统: [10:30] 今天感觉状态不错 → 写入 2026-01-21.md
```

### 场景2: 查看原文

需要回顾某天的原始想法时，打开对应日期的碎片文件即可。

### 场景3: 二次分析

基于一周或一月的碎片原文，进行深度模式分析。

## 📊 与复盘的关系

| 类型 | 碎片存档 | 日报复盘 |
|------|---------|---------|
| **内容** | 原文记录 | 深度分析 |
| **时机** | 实时追加 | 晚上召唤 |
| **形式** | 时间+原文 | 6模块结构 |
| **价值** | 完整性 | 洞察性 |
| **用途** | 二次分析 | 当日复盘 |

## 💡 最佳实践

1. **随时记录**: 有想法就记录，不要等到晚上
2. **原汁原味**: 想到什么就记什么，不要修饰
3. **定期回顾**: 每周看一次碎片，寻找模式
4. **对比分析**: 对比碎片和日报，看小北如何加工

## 🔐 隐私保护

- 所有碎片存储在本地
- 只有你能看到原文
- 可以选择性删除敏感内容

## 📈 数据价值

长期积累的碎片存档可以：

- 识别思维模式
- 追踪情绪变化
- 发现行为规律
- 见证认知进化

---

**记住**: 碎片是原材料，复盘是加工品。两者结合，才能看清完整的自己。
